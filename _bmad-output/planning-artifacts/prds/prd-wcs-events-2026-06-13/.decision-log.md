# Decision Log — Tulsa WCS Events PRD

_Created: 2026-06-13_

---

## Decisions

### DL-1 — Two-phase release model
**Decision:** Separate Deploy (Phase 1) from Community Launch (Phase 2).
**Rationale:** Jason is new to the community and aware of community politics. Jubilee needs to do relational groundwork before any public announcement. Phase 1 goal is a URL Jubilee can preview and approve. Phase 2 timing is owned entirely by Jubilee.
**Captured in PRD:** Section 4.

### DL-2 — Trust-first moderation for MVP
**Decision:** All submissions go live immediately; no review queue.
**Rationale:** Tulsa WCS is small enough that bad actors self-correct via community reputation. Defers complexity.
**Captured in PRD:** Section 11.

### DL-3 — Four confirmed views
**Decision:** Tonight (home), Browse, Archive, Submit (footer only).
**Rationale:** Confirmed in brainstorming Phase 2 mind-mapping.
**Captured in PRD:** Section 7.

### DL-4 — Organic submission metric deferred
**Decision:** No automated metric for counting non-seeded submissions in MVP. Manual proxy only.
**Rationale:** Google Form submissions don't auto-report counts to any trackable surface.
**Captured in PRD:** OQ-2.

### DL-5 — Mailing list provider assumed Mailchimp
**Decision:** Mailchimp free tier assumed unless Jason specifies otherwise. Constraint: no server-side code.
**Captured in PRD:** FR-18, OQ-1.

### DL-6 — Site name and domain deferred
**Decision:** Jason and Jubilee decide together. GitHub Pages URL fine for Phase 1.
**Captured in PRD:** OQ-4.

### DL-7 — Mailing list provider is an implementation detail
**Decision:** Removed from PRD. Provider choice is deferred to build; PRD only requires no server-side code and a free tier sufficient for early subscriber counts.

### DL-8 — Date filter interaction model is an implementation detail
**Decision:** FR-7 states the functional requirement (date filtering, client-side) only. How the filter control is implemented is delegated to UX and architecture.

### DL-9 — PRD finalized
**Date:** 2026-06-13. Status set to final. Formal finalize pass complete (reconciliation, rubric review, structural polish, prose polish).
